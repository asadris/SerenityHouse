# **Serenity House Analytics – Proof of Concept (PoC)**  
*A portfolio‑ready demonstration of modern data architecture, synthetic data generation, and Power BI analytics for a 45‑bed transitional recovery residence.*

---

## 🌟 **Project Overview**

This Proof of Concept (PoC) demonstrates what a modern, unified data and analytics system could look like for **Serenity House**, a nonprofit transitional recovery residence providing housing, stability, and wrap‑around services for individuals rebuilding their lives.

The PoC showcases how clean data modeling, synthetic datasets, and interactive Power BI dashboards can transform program visibility, operational decision‑making, and outcome measurement. It is intentionally **isolated**, **time‑boxed**, and **designed for demonstration**, not production.

---

## 🎯 **Purpose of the PoC**

This PoC was created to:

- Validate the feasibility of a modernized analytics ecosystem  
- Demonstrate dashboards for engagement, stability, finances, employment, and outcomes  
- Test assumptions about workflows, data structures, and reporting needs  
- Surface unknowns early before full implementation  
- Provide a shared vision for leadership, staff, and funders  
- Serve as a polished, portfolio‑ready artifact showcasing architectural thinking and analytics craftsmanship  

This PoC is not a prototype of the final system — it is a **guided preview** of what a full analytics modernization could deliver.

---

## 📦 **What This PoC Includes**

- A normalized **operational schema** representing residents, beds, services, incidents, employment, finances, and outcomes  
- A **synthetic data generator** (Python) modeling realistic program behavior  
- A lightweight **warehouse-style analytics model**  
- Five **Power BI dashboards** demonstrating key program insights  
- Architecture diagrams and documentation  
- Reproducible folder structure and example ETL logic  
- A clean, professional GitHub repository suitable for portfolio presentation  

---

## 🚫 **What This PoC Does *Not* Include**

To maintain clarity and scope discipline, this PoC intentionally excludes:

- Production pipelines or deployment automation  
- Real participant data  
- Authentication, security hardening, or governance layers  
- Integration with operational systems  
- Long‑term maintenance workflows  
- Full enterprise ETL orchestration  

These components belong in the future production implementation.

---

## 🏗 **Architecture Summary**

The PoC follows a simple, transparent flow:

```
Synthetic Data → Operational Schema → Analytics Model → Power BI Dataset → Dashboards
```

### Key Design Principles
- **Normalization first** to ensure data integrity  
- **Synthetic data** to protect confidentiality while enabling realistic modeling  
- **Separation of concerns** between operational and analytical layers  
- **Power BI semantic modeling** for reusable measures and consistent reporting  
- **Lightweight, reproducible structure** suitable for rapid iteration  

---

## 📊 **Dashboards Included**

### **1. Resident Engagement**
Tracks participation in services, meetings, and program activities.

### **2. Behavior & Stability**
Visualizes incidents, drug tests, curfew compliance, and stability indicators.

### **3. Financial Responsibility**
Shows resident fees, payments, balances, and financial progress.

### **4. Employment Progress**
Monitors job attainment, hours worked, wages, and employment stability.

### **5. Program Outcomes**
Summarizes completion rates, housing stability, employment outcomes, and long‑term success indicators.

Each dashboard is built on a shared semantic model to ensure consistent metrics across the PoC.

## ▶️ **How to Run the PoC Locally**

1. Clone the repository  
2. Run the Python synthetic data generator  
3. Execute the SQL scripts to build the schema  
4. Load the generated data into your SQL environment  
5. Open the Power BI files in the `/powerbi` folder  

No production infrastructure or pipelines are required.
